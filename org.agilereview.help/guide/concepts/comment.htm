Comment concept

Definition of "comment" in context of the AgileReview plugin

A comment is a set of information regarding a file or one or more lines of code of a file that is part of a project of the Eclipse workspace. It is related to exactly one file and exactly one review. A comment consists of the following information:

  • ID - the comment's ID (unique in context of an author and a review)
  • Author - the comment's author
  • Status - the comment's actual status (open, closed)
  • Priority - the comment's priority (high, medium, low)
  • Recipient - the person that this comment is addressed to
  • Description - a short description of the issue marked by this comment
  • Replies - a list of replies that were added by the author and other reviewers
A comment can always be identified by the triple (review ID,author,ID).

Code line bound comments

A comment that is related to one or more lines of code of a file is called "line bound". It will be highlighted in the editor containing the file and show up in the table of the "Comments Summary" view.

Global comments

A comment that is solely related to a file is called "global". It will not be highlighted in the editor containing the file, but show up in the table of the "Comments Summary" view. Global comments will be created for file types that are not supported (XML, Java).
